C++ 程序编译过程 编译过程分为四个过程：编译（编译预处理、编译、优化），汇编，链接。 编译预处理：处理以 # 开头的指令； 编译、优化：将源码 .cpp 文件翻译成 .s 汇编代码； 汇编：将汇编代码 .s 翻译成机器指令 .o 文件； 链接：汇编程序生成的目标文件，即 .o 文件 链接分为两种： 静态链接：代码从静态链接库中拷贝到可执行程序，程序被执行时，代码会被装入到该进程的虚拟地址空间； 动态链接：代码被放到动态链接库或共享对象的某个目标文件中，链接程序只是在最终的可执行程序中记录了共享对象的名字等一些信息。在程序执行时，动态链接库的全部内容会被映射到运行时相应进行的虚拟地址的空间。 两种连接方式的优缺点： 静态链接：浪费空间，更新困难，链接时同一个库可能链接多次，包含多个副本，库更新整个程序需重新编译。 动态链接：节省内存、更新方便，动态库内存中只有一份拷贝，但程序运行时，每次都需要链接，会产生性能损失。 C++内存管理与static关键字 C++ 内存分区 栈：存放函数的局部变量、函数参数、返回地址等，由编译器自动 ...
