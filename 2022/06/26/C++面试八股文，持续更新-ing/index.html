C++ 程序编译过程

编译过程分为四个过程：编译（编译预处理、编译、优化），汇编，链接。

  • 编译预处理：处理以 # 开头的指令；

  • 编译、优化：将源码 .cpp 文件翻译成 .s 汇编代码；

  • 汇编：将汇编代码 .s 翻译成机器指令 .o 文件；

  • 链接：汇编程序生成的目标文件，即 .o 文件

链接分为两种：

  • 静态链接：代码从静态链接库中拷贝到可执行程序，程序被执行时，代码会被装入到该进程的虚拟地址空间；
  • 动态链接：代码被放到动态链接库或共享对象的某个目标文件中，链接程序只是在最终的可执行程序中记录了共享对象的名字等一些信息。在程序执行时，动态链接库的全部内容会被映射到运行时相应进行的虚拟地址的空间。

两种连接方式的优缺点：

静态链接：浪费空间，更新困难，链接时同一个库可能链接多次，包含多个副本，库更新整个程序需重新编译。

动态链接：节省内存、更新方便，动态库内存中只有一份拷贝，但程序运行时，每次都需要链接，会产生性能损失。

C++内存管理与static关键字

C++ 内存分区

  • 栈：存放函数的局部变量、函数参数、返回地址等，由编译器自动分配和释放；
  • 堆：new/malloc动态申请的内存空间，程序员控制分配和释放；
  • 全局区/静态存储区（.bss 段和 .data 段）：存放全局变量和静态变量；
  • 常量存储区（.data 段）：存放的是常量，不允许修改，程序运行结束自动释放；
  • 代码区（.text 段）：存放编译后的二进制代码文件，不允许修改，但可以执行。

注：存储区在内存中的分布（地址由低到高）：text 段 --> .data 段 --> .bss 段 --> 堆 -->-unused --> 栈 --> env

栈和堆的区别

存储 申请效率
栈 连续空间，向低地址扩展，分配失败会导致栈溢出 系统自动分配，申请效率高
堆 非连续空间，首地址存放长度，存储方式类似链表 主动申请主动释放，效率低，易产生内存碎片

static关键字

全局变量：具有全局作用域，只需在一个文件中定义即可作用于所有文件，其他文件需要用extern再次声明；

  • 静态全局变量：具有文件作用域，只能在当前文件中使用；
  • 静态局部变量：具有局部作用域，只需要初始化一次且只对当前函数可见，生命周期持续到程序结束。
  • 静态成员变量：
    • 静态成员不属于某个具体对象，而是被所有对象共享
    • 不能在构造函数的初始化列表中直接进行初始化
    • static成员变量在类内部声明，只可在外部初始化，初始化不需要加static，如： int Date::day = 14;
  • 静态成员函数：
    • 不可以调用类的非静态成员变量
    • 静态成员函数不包含指向具体对象的this指针

voliate关键字

对于普通变量，编译器通常会将变量的值存储在一个寄存器，后续使用会直接在寄存器中取数。voliate通常用来修饰随时变化的量，编译器不会进行优化，每次都会从该地址处取数。

参数可以同时被const和voliate修饰吗？

可以的，如：只读的状态寄存器，用来修饰程序无法改变但数据实时变化的量。

voliate关键字的应用场景

  • 中断服务程序中修改的供其它程序检测的变量需要加volatile；
  • 多任务环境下各任务间共享的标志应该加volatile；
  • 存储器映射的硬件寄存器通常也要加volatile说明，因为每次对它的读写都可能由不同意义。
